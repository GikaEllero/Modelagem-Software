Ordem de Compra

UMLUseCase
Untitled :: Model1 :: Ordem de Compra

Description

Fluxo Principal 1. O usuário avalia as ações disponíveis (Fluxo Alternativo - 01) 2. O usuário escolhe o tipo de ordem de compra e a quantidade (Fluxo Alternativo - 02) 3. A ordem de compra é enviada para a corretora (Fluxo Alternativo - 03) 4. A corretora envia a ordem de compra para a B3 5. A B3 retorna para a corretora que a ordem foi aprovada (Fluxo Alternativo - 05) 6. A corretora notifica o usuário da aprovação

Fluxo Alternativo - 03 3.1. O usuário pode visualizar o status da ordem de compra 3.2. O usuário pode cancelar ou modificar as informações da ordem de compra 3.3. As modificações são enviadas a corretora 3.4. Retorna ao Fluxo Principal 4

Fluxo Alternativo - 05 5.1. A B3 retorna para a corretora que a ordem foi reprovada 5.2. A corretora notifica o usuário da reprovação

Fluxo Alternativo - 02 2.1. O usuário escolhe o tipo de ordem de gatilho 2.2. O usuário escolhe o valor no qual a ação será comprada 2.3. Se atingir o valor desejado, retorna ao Fluxo Principal 3

Fluxo Alternativo - 01 1.1. Se o usuário já tiver uma ordem de compra para ação visualizada aparece uma notificação
